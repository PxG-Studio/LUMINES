# SLATE - Subsystem Layout & Theming Engine

**SLATE** is the centralized design system and token management layer for all WISSIL subsystems.

## Purpose

SLATE serves as the single source of truth for:
- **Design Tokens**: Colors, typography, spacing, shadows
- **Theme Configuration**: Light/dark modes, system themes
- **Visual Consistency**: Standardized design language
- **Component Styling**: Base styles and utilities

## Architecture

### Token Structure

```typescript
slateTokens
├── colors
│   ├── landing (primary, secondary, accent, gradient)
│   ├── slate (primary, secondary, accent, gradient)
│   ├── ignition (primary, secondary, accent, gradient)
│   ├── spark (primary, secondary, accent, gradient)
│   ├── ignis (primary, secondary, accent, gradient)
│   ├── waypoint (primary, secondary, accent, gradient)
│   ├── nocturna (dark, midnight, twilight, dusk, dawn)
│   └── neutral (50-900)
├── typography
│   ├── fontFamily (sans, mono, display)
│   ├── fontSize (xs - 9xl)
│   ├── fontWeight (thin - black)
│   ├── lineHeight (none - loose)
│   └── letterSpacing (tighter - widest)
├── spacing (px, 0 - 96)
├── borderRadius (none - full)
├── shadows (sm - 2xl, glow)
├── animation (duration, easing)
├── breakpoints (sm - 2xl)
└── zIndex (hide - tooltip)
```

### Network Topology

- **Location**: Helios Compute (192.168.86.115)
- **Port**: 3001
- **Protocol**: HTTPS
- **Auth**: nocturnaID with Designer role
- **Access**: Authenticated users only

## Usage

### Importing Tokens

```typescript
import { slateTokens, getSystemColors } from '@/tokens/slate.tokens';

// Get all tokens
const colors = slateTokens.colors;
const spacing = slateTokens.spacing;

// Get system-specific colors
const landingColors = getSystemColors('landing');
```

### Tailwind Integration

All SLATE tokens are automatically available in Tailwind:

```tsx
// Color classes
<div className="bg-slate-primary text-white" />
<div className="text-gradient-landing" />

// Spacing
<div className="px-4 py-8 gap-6" />

// Typography
<h1 className="text-7xl font-bold" />
```

### CSS Variables

SLATE tokens can be exported as CSS custom properties:

```css
:root {
  --color-landing-primary: #FFD700;
  --spacing-4: 1rem;
  --font-size-7xl: 4.5rem;
}
```

## Token Management

### Adding New Tokens

1. Update `src/tokens/slate.tokens.ts`
2. Add to appropriate category
3. Update Tailwind config if needed
4. Document in SLATE page
5. Run validation tests

### Naming Convention

Follow this pattern for consistency:
- **Colors**: `system-category` (e.g., `landing-primary`)
- **Spacing**: Numeric scale (e.g., `4`, `8`, `12`)
- **Typography**: T-shirt sizing (e.g., `xs`, `sm`, `lg`)
- **Effects**: Descriptive (e.g., `glow`, `shimmer`)

### Token Validation

```typescript
// Validate color contrast
checkContrast(foreground, background) >= 4.5

// Validate spacing scale
isMultipleOf(spacing, 4) === true

// Validate font scale
fontScale.follows(perfectFourth) === true
```

## Best Practices

1. **Always use tokens**: Never hardcode colors or spacing
2. **System colors**: Use appropriate system colors for context
3. **Responsive scales**: Utilize breakpoint tokens
4. **Accessibility**: Maintain WCAG AA+ contrast ratios
5. **Consistency**: Stick to the defined scale

## Performance

- **Token file size**: < 5 KB (minified)
- **Tree-shakeable**: Unused tokens removed in production
- **CSS generation**: Tailwind JIT for minimal output
- **Cache-friendly**: Tokens rarely change
